## ⚙️ Обробка даних

### Data Dictionary

| Змінна | Опис | Тип |
|---|---|---|
| `Price_main` | Загальна вартість об'єкта (USD) | Числовий |
| `area` | Загальна площа квартири (м²) | Числовий |
| `build_year` | Рік побудови будинку | Числовий |
| `building_age` | Вік будинку (2026 − рік побудови) | Числовий |
| `Price_for_m2` | Ціна за м² (USD) | Числовий |
| `type_build` | Категорія житлового фонду | Категоріальний |
| `district` | Адміністративний район | Категоріальний |

### Кроки обробки

1. **Видалення зайвих колонок** — видалено `Id` та `Address` як нерелевантні для аналізу.
2. **Оптимізація типів** — `build_year` та `Price_for_m2` приведено до `Int64`.
3. **Фільтрація** — видалено записи, де `district == 'Київ'` (без конкретного району).
4. **Feature Engineering** — розраховано `building_age` та застосовано класифікацію `type_build`:
   - `New building` — до 5 років
   - `New housing stock` — від 5 до 25 років
   - `Old housing stock` — понад 25 років
5. **Фільтрація викидів** — видалено аномальні значення ціни та площі.

```python
def type_buaild(building_age):
    if building_age < 5:
        return 'New building'
    elif building_age <= 25:
        return 'New housing stock'
    else:
        return 'Old housing stock'

# Вік будинку
df_clear['building_age'] = 2026 - df_clear['build_year']
df_clear['type_build'] = df_clear['building_age'].apply(type_buaild)

# Фільтрація викидів
df_no_outliers = df_clear[
    (df_clear['Price_main'] >= 10000) &
    (df_clear['Price_main'] <= 2000000) &
    (df_clear['area'] >= 15)
]

# Статистика по районах
district_stats = df_no_outliers.groupby('district').agg({
    'Price_for_m2': 'mean',
    'Price_main': 'median'
})
```

## 💡 Ключові інсайти

**1. Районна сегментація ціни за м²**

Рейтинг районів за медіанною ціною (дані дашборду):

| # | Район | Медіанна ціна, $/м² |
|---|---|---|
| 1 | Печерський | ~$3 800 |
| 2 | Шевченківський | ~$2 800 |
| 3 | Голосіївський | ~$2 300 |
| 4 | Подільський | $1 667 |
| 5 | Солом'янський | ~$1 600 |
| 6 | Оболонський | ~$1 550 |
| 7 | Дніпровський | ~$1 450 |
| 8 | Дарницький | ~$1 350 |
| 9 | Святошинський | ~$1 300 |
| 10 | Деснянський | ~$1 100 |

Різниця між Печерським та Деснянським районами складає **більш ніж 3×**.

**2. Залежність площі від ціни**

Scatter plot демонструє чітку лінійну залежність між площею та загальною вартістю. У масовому сегменті (до 100 м²) залежність стабільна, тоді як у преміум-сегменті (100–400 м²) дисперсія значно зростає — локація та поверх стають визначальними факторами.

**3. Фактор віку будинку**

Категоризація `type_build` дозволила виявити суттєвий ціновий розрив між новобудовами та старим фондом у межах одного району. Новобудови (`New building`) стабільно дорожчі навіть у спальних районах — попит на нове житло не залежить від локації.

## 📝 Висновки

Ринок нерухомості Києва демонструє виражену сегментацію, де ключовими драйверами ціноутворення виступають географія (район) та площа об'єкта. Найсильнішим предиктором ціни є локація — різниця медіанної вартості м² між Печерським та Деснянським районами сягає 2,5×. Загальна площа лінійно корелює з вартістю, проте у сегменті великогабаритного житла (понад 100 м²) спостерігається значна дисперсія цін. У цьому преміальному сегменті визначальними стають додаткові характеристики об'єкта, які можна детальніше дослідити за допомогою інтерактивних фільтрів дашборда (поверх, рік будівництва).
